#### Quartz

What is quartz?
- Quartz is a static Site Generator.
- It Converts Markdown to HTML so that it can be rendered as the webpage.
- Quartz is a separate tool, that works well with Obsidian.
- It is not a plugin.
- Quartz can deploy any Markdown notes, it is not compulsory to be obsidian.

Generally, the three Tools needed are:
i. Markdown Editor : Obsidian
ii. Static Site Generator : Quartz
iii. Version Control : Git and Github
iv. Hosting Provider : Github Pages or Netlify

###### 2. Quartz requires at least Node v22 and npm v10.9.2 to function correctly

What they are
i. <span style="color:rgb(255, 0, 255)">Node.js</span>
    - Node.js is a <span style="color:rgb(255, 255, 0)">runtime environment</span> that allows you to run JavaScript outside the browser (on your computer/server).
    - It’s what <span style="color:rgb(255, 255, 0)">powers</span> modern static site generators (like <span style="color:rgb(255, 255, 0)">Quartz</span>) and many developer tools.
        
ii. <span style="color:rgb(255, 0, 255)">npm</span> (Node Package Manager)    
    - npm is a <span style="color:rgb(255, 255, 0)">package manager</span> that comes with Node.js.
    - It’s used to <span style="color:rgb(255, 255, 0)">install</span>, <span style="color:rgb(255, 255, 0)">update</span>, and <span style="color:rgb(255, 255, 0)">manage</span> libraries (called **packages**) that your project needs.
    - Example: Quartz uses npm to <span style="color:rgb(255, 255, 0)">fetch</span> <span style="color:rgb(255, 255, 0)">dependencies</span> like Markdown parsers, build tools, plugins, etc.

Installing them in Linux
a. Install nvm (Node Version Manager)
```bash
//Installation
curl -fsSL https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash

//Reload the Shell, so that it recognises nvm
source ~/.bashrc
```

b. Install node.js v22
```bash
//Installation
nvm install 22

//Make it default, now using node v22
nvm use 22
nvm alias default 22

```

c. Verify Quartz Prerequisites
```bash
node -v //v22.19.0

npm -v //10.9.3
```


Continue Setting up Quartz
a. Change directory to the Quartz folder(Folder you cloned)
```bash
cd  quartz
```

b. Install the dependencies needed
```bash
npm i //The dependencies start to be installed
```

> Meaning: Install all dependencies listed in your project’s `package.json`.
   What it does:
    1. Looks at your project’s `package.json` file.
    2. Downloads all required libraries (dependencies).
    3. Puts them into the `node_modules` folder.
    4. Creates/updates a `package-lock.json` file to lock versions.
    . 
        Example: If Quartz needs libraries like `vite`, `remark`, or `esbuild`, `npm i` will fetch them for you.

c. Quickly grab the Quartz tool and build me a new blog skeleton
```bash
npx quartz create
```

> Meaning : Run the Quartz “create” command without installing it globally.
    What it does:
    1. `npx` downloads and runs the Quartz CLI tool **temporarily** (so you don’t have to install it permanently).
    2. `quartz create` starts a new Quartz project.
    3. It will ask you some setup questions (like folder name, theme, etc.). 
    4. Creates a starter Quartz blog in the chosen directory.
    . 
         Think of `npx quartz create` as: “Get me Quartz right now and start a new site,” without cluttering your system.


###### 3. Choose How to Initialize Your content

i. Empty Quartz (●)
- Starts with a **blank blog** just the Quartz structure, no notes.
- You’ll need to copy/write your own Markdown notes into `/quartz/content`.
- Best if you want a **fresh start**.

ii. Copy an existing folder (○)
- Lets you pick a folder (like your **Obsidian vault** ) and Quartz will **copy all notes** into its `/content` folder.
- Good if you already have notes you want to publish.
- Downside: The Quartz copy is separate if you update your Obsidian vault later, it won’t auto-sync unless you copy again.

iii. Symlink (○)
- Creates a **symlink (shortcut link)** from Quartz’s `/content` to your Obsidian vault folder.
- This means any edits in Obsidian → instantly reflect in Quartz (no need to copy files).
- Best if you want **live sync between Obsidian and Quartz**.
    

Recommendation for you (since you want to blog from **Obsidian notes**):  
Choose <span style="color:rgb(255, 255, 0)">Symlink</span> so Quartz always reads directly from your Obsidian vault.
