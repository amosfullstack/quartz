## Description

REST API Design, Development & Management Course. This course focuses on designing and developing RESTful APIs using best practices and aims to teach you how to create robust, maintainable, and scalable APIs. Rather than focusing on API coding, this course emphasizes RESTful design principles and architecture. This course covers all the important aspects related to API design, development, and management. The best practices, design patterns, challenges, recommendations, and options discussed in this course are either created through analyzing how popular API providers like Facebook, Twitter, Capital One, etc. build and manage APIs, or are drawn from the author’s personal experiences.

### What you will learn in this course:

- Design and develop RESTful APIs using REST best practices and constraints
- Establish procedures for API security, versioning, lifecycle management, documentation, and other important aspects
- Writing specifications in Swagger2.0/OAI format in YAML format
- Create an API management strategy for your organization
- Leverage some of the popular API management platforms to build API proxies (APIGEE, IBM API Connect, Mulesoft Anypoint)

### This course is suitable for people who:

- Technologists interested in learning where, when, and how to use RESTful services in applications.
- Web, mobile, IoT application developers looking to build RESTful services on the backend.
- Solution architects who are interested in learning how to use “API management platforms” such as Apigee/Mulesoft/IBM API Connect.
- IT administrators who aspire to become architects.
- This course does not teach students how to code Node JS. It uses NodeJS to demonstrate the implementation of concepts discussed in the lectures.
- This course is not suitable for students who are just looking to learn how to code REST APIs.

### REST API Design Development & Management Course Specifications

### REST API Design Development & Management Course Prerequisites

- Any one modern programming language such as Java or Node JS or Go or PHP
- Familiarity with web application architecture
- Idea on XML or JSON formats
- Understanding of the concept of service
