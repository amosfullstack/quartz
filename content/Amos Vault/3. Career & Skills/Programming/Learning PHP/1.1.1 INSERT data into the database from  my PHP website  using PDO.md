## **Overview**
- Make clear the concept of inserting the data into the database from the PHP website using PDO.

## **Steps for data insertion**
###### 1. Create a new php script to handle the form data
- The file file may be named formhandler.php. If it is pure php, there should be no closing tag.

###### 2. Check if the user submited the form the right way.
- Here, im checking if the user actually submited the form using the correct POST method ie the right way. Then i will allow the formhandler.php page to be accessed. If not then i will send the user back to the front page.
```php
if($_SERVER[REQUEST_METHOD]==="POST"){
    //Continue here with the normal code operations
}
else{
   //Send the user back to the home page
   header("Location: index.php"); //Navigate the directories if required
}
```

> So basically the user can not access the <span style="color:rgb(255, 0, 255)">formhander.php</span> file without first actually submitting the form.

###### 3. Grab the User data
- (Inside the if statement)
- So, if the user accessed the page correctly, now <span style="color:rgb(255, 255, 0)">grab</span> the user data.
- Create the <span style="color:rgb(255, 255, 0)">varibles</span> that will store the grabbed data from the form.
- Inside the post superglobal, i am going to <span style="color:rgb(255, 255, 0)">reference to the name attribute</span> that i used in html.
```php
$username = $_POST["username"];
$pwd = $_POST["password"];
$email = $_POST["email"];
```

> [!Note] 
> Use the htmlspecialchars() method for sanitization, when you want to output the data to the browser.
> ie: 
> ```php
>   echo htmlspecialchars($username);

###### 4. Create a try catch block to handle the Errors of the SQL Query.
- (Inside the if statement, after grabbing the data)
- Make sure you connect to the file that has the database connection details before writing the Query ie: <span style="color:rgb(255, 255, 0)">dbhandler.php</span> | <span style="color:rgb(255, 255, 0)">dbconnection.php</span>

```php
try{
 require_once "dbconnection.php";  //Navigate to the directory containing the file if required.
 //Using require is the same as writing the entire code contained in that script

}
catch(PDOException $e){
   die("Query failed: ". $e->getMessage());
}
```

> At <span style="color:rgb(255, 255, 0)">require_once</span> : We can also use these. All of them basically do the same thing with slight variation.
>                           - <span style="color:rgb(255, 148, 148)">require</span> : Does what all others do but instead of throwing a warning, it will run an error. It is going to show a fatal error saying. I cant find this file so stop everything from running.
>                          
>                           - <span style="color:rgb(255, 148, 148)">include</span> : Include the given file, if the file cant be found then gives a small warning. Saying i cant find the file.
>                           
>                           - <span style="color:rgb(255, 148, 148)">include_once</span> : This does what include does but also checks if the file was already included inside the sccript. And then throws a warning.

###### 5. Create the SQL query
- (after <span style="color:rgb(255, 255, 0)">require_once</span>).
```php
$query = "INSERT INTO users(username,password,email) VALUES ($username,$pwd,$email);";
//This is also correct, we have directly used the variables.
```

```php
$query = "INSERT INTO users(username,password,email) VALUES ($?,?,?);";
//The best way is to use place holders for varibales ,then later on, user data will be bound here.
```

###### 6. Create a prepared statement and submit the query
- (after creating the query)
- Now <span style="color:rgb(255, 255, 0)">create the prepared</span> statement and submit the query to it to be executed in the database.

```php
$stmt=$pdo->prepare($query);
```
From the code snippet:
> <span style="color:rgb(255, 148, 148)">$stmt </span>- Is the name given to the statement we are preparing.
> <span style="color:rgb(255, 148, 148)">$pdo->prepare()</span> - In the PDO object i created i am calling the method called prepare()
> <span style="color:rgb(255, 148, 148)">$query </span>- This is now the submission of the query i want to be excecuted.

###### 7. Submit the user data to the statement
- (after creating the prepared statement)
- <span style="color:rgb(255, 255, 0)">Submit the user data</span> to the statement using an array.
```php
$stmt->execute([$username,$pwd,$email]);
```
From the code snippet:
> <span style="color:rgb(255, 148, 148)">$stmt->execute()</span> - Isthe method. It **actually runs** the prepared SQL query against the database.
> - If you provided placeholders (like `?` or `:name`), you either:
    1. Pass values during `execute([...])`, OR
    2. Bind values first with `bindParam()` / `bindValue()` and then just call `execute()`.

So, Alternatively. Using <span style="color:rgb(255, 0, 255)">name parameters</span> (instead of just question marks) and then binding them to varibales.
```php
//Create the sql query
$query = "INSERT INTO users($username,$password,$email) VALUES(:username,:pwd,:email);";

//Create a prepared statement
$stmt = $pdo->prepare($query);

//Bind the name parameters to the variables using bind param method
$stmt->bindParam(":username",$username);
$stmt->bindParam(":pwd",$password);
$stmt->bindParam(":email",$email);

//Finally Execute the Statement
$stmt->execute();

//Close the connection
$pdo = null;
$stmt = null;

//Return the user to the home page
header("Location: ../index.php");

//Kill the script
die();
```

###### 8. Manually Close the Statement and the connection
- This is not the must to do it, because is is ussually done automatically
- It is considered the best practice because you free the resources early
```php
$pdo = null;  //Set the pdo database connection to null
$stmt = null; //Set the prepared statemt also to null

//Send the user back to front page and kill this script
header("Location: ../index.php");

die(); //Write the die method to stop any execution OR
exit(); //Just use exit
```
## **New Terms**
i. <span style="color:rgb(255, 255, 0)">die()</span> :  Is a PHP function that **stops the execution** of a script immediately.
            : You can also pass a message to display before stopping.

ii. <span style="color:rgb(255, 255, 0)">htmlspecialchars()</span> : Converts **special characters** into **HTML entities** so they are displayed safely in the                                      browser.    
                                    : This prevents security issues like **XSS (Cross-Site Scripting)**.
                                    
iii. <span style="color:rgb(255, 255, 0)">require_once</span> : Used to include a PHP file into another file.
                            : Similar to `require`, but it **only includes the file once** (even if you call it multiple times).
                            : Prevents redeclaration errors.
