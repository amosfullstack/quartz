## **Overview**
- In this case we i am going to learn how to connect the php codebase to the backed mysql database using (PHP Data Object) PDO Connection type. 
- Peforming SQL query Operations to the database from the codebase such as : INSERT, DELETE, UPDATE etc.
- PDO is a type of connection that if flexible to many database systems, it is not fixed to one type of database. One php codebase can be connection to multiple types of databases in the backend.
## **Steps for PDO Database Connection in PHP**

###### 1. **Create a database handler file**: ie dbconnect.php

###### 2. Create a dsn(data source name)

- A <span style="color:rgb(255, 255, 0)">dsn(data source name)</span> is a piece of code that contains info about the database, ie: database driver,kind of the host and database name.
  ```php
  $dsn = "mysql:host=localhost;dbname=myfirstdatabase";
```

From the above code:
> i. <span style="color:rgb(255, 148, 148)">mysql:</span> - Is the name of the driver that our database is using.
> ii. <span style="color:rgb(255, 148, 148)">host=localhost;</span> - Specifies the name of the host our database is located. In this case we are using    localhost because the database is on the xampp local server.
> iii. <span style="color:rgb(255, 148, 148)">dbname=myfirstdatabase;</span> - Tells the codebase what name is given to the database that we are trying to connect to.

###### 3. **Provide the database Username and Password**

- The database contain the username and password. It should be provided inside the script for login authentication.
```php
$dbusername = "root";
$dbpassword = "root"; //It can sometimes be empty depending on the database settings.
```

###### 4. Establish the connection using the pdo object.

```php
$pdo = new PDO($dsn,$dbusername,$dbpassword);
```
- The code can be refactored, Use the try and catch block to handle errors.
```php
try(){
   $pdo = new PDO($dsn,dbusername,$dbpassword);
   $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION); //If we get an Error then throw an exception
} catch(PDOException $e){//Catch the exception and name it variable $e

  echo "Connection failed: ". $e->getMessage();
  //Display the above message and concatenate it with the specific Error
}
```

From the code snippet above:
> i. <span style="color:rgb(255, 148, 148)">$pdo = new PDO()</span> - Instantiates a new PDO object inside our existing class inside our  PHP language, PDO is a prebuilt class that its object is instantiated when needed.
>                                    - The pdo object is given parameters that are required to establish connection ie: data source name, dbusername and dbpassword.
> ii. <span style="color:rgb(255, 148, 148)">$pdo->setAttribute()</span> - We are calling a method(pointing to a method), this method exists in our newly created $pdo object.
>                                         - Is configuring how **PDO (PHP Data Objects)** should handle errors when something goes wrong with your database operations.
>                                         - The `setAttribute()` method allows you to configure PDO’s behavior by setting attributes.
>                                         - <span style="color:rgb(255, 255, 0)">PDO::ATTR_ERRMODE</span> : This is the **error handling attribute** in PDO.
>                                                                                  : t controls how PDO reports errors when a database issue occurs (e.g., invalid query, connection problem).
>                                                                                  
>                                         - <span style="color:rgb(255, 255, 0)">PDO::ERRMODE_EXCEPTION</span> : This tells PDO to **throw an exception**  (                                                                 `PDOException`) whenever an error occurs.
>                                                                                             : Instead of just returning `false` or giving a silent error, it forces the script to raise an exception.
>                                         - This is useful because: You can catch the exception with `try...catch`.
>                                         - It makes debugging easier since you get the actual error message, file, and line number.
>                                         - That line tells PDO **“if something goes wrong, don’t fail silently throw an exception so I can handle it properly.”**


Other possible error modes (For context):
i. <span style="color:rgb(255, 255, 0)">PDO::ERRMODE_SILENT</span> (default)
    - Fails silently, only setting error codes. You’d have to manually check `$pdo->errorInfo()`.
        
ii. <span style="color:rgb(255, 255, 0)">PDO::ERRMODE_WARNING</span>
    - Emits a PHP warning, but script continues.

iii. <span style="color:rgb(255, 255, 0)">PDO::ERRMODE_EXCEPTION</span> (Our case)
    - Throws an exception and stops execution unless caught.

#### **New Terms**
1. <span style="color:rgb(255, 148, 148)">Exception</span> : An **error event** that stops normal program flow and can be "caught" and handled in a                             controlled way.
                         : A special type of error that you can **catch and handle** instead of letting your program crash unexpectedly.

### Entire Script to Establish Connection
```php
<?php
//Create a data source name
$dsn = "mysql:host=localhost;dbname=myfirstdatabse";

//Provide db username and password
$dbusername = "root";
$dbpassword = "root";

//Establist connection, handle Errors
try(){
   $pdo = new PDO($dsn,$dbusername,$dbpassword);
   //Set the attribute for Grabbing exception
   $pdo->setAttribute(PDO::ATTR_ERRMODE,ERRMODE_EXCEPTION); //Throw an exception
}
catch(PDOException $e){
  echo "Connection failed: ". $e.getMessage();
}
?>
```

###### 5. **Perform the SQL Operations to Manipulate the Database**
- After the database connection is successful <span style="color:rgb(255, 255, 0)">sql operations</span> can be performed to manipulate data inside the database.
- I am going to be using <span style="color:rgb(255, 255, 0)">prepared statements</span> , i can insert data into the the database without using prepared statements but that is not secure.
- From the form data is submitted to the databse using the <span style="color:rgb(255, 255, 0)">POST</span> Method. To grab  data from the database we use the <span style="color:rgb(255, 255, 0)">GET</span> method.
- The <span style="color:rgb(255, 255, 0)">name</span> attribute is the one being used in php to referrence the elements inside the html form.
#### New Terms
i. <span style="color:rgb(255, 148, 148)">A prepared statement </span>: Is a way to safely and efficiently run SQL queries in databases using placeholders for values instead of directly inserting them into the query.
                                        : Instead of writing SQL with raw values, you write it with **placeholders** (like `?` or `:name`), and then you **bind values** separately.
  
> Basically, prepared statements prevent SQL injection at the user inputs. Example at the user input, the attacker may write the SQL query that goes in and deletes the database. So I prevent the user from writing SQL code directly at the websites input using prepared statements.

How prepared Statements work:
- The sql query is sent to the database first.
- Then we bind data submitted by the user.
- The data is then sent to the database After wards.


- Why We Use them?
a. **Security (Prevents SQL Injection)**
    - If you insert user input directly into SQL, attackers can inject malicious SQL code.
    - Prepared statements keep data separate from the SQL logic.
    - The database knows which part is SQL and which part is data.
        
b. **Performance**
    - The SQL query structure is sent to the database **once** and compiled.
    - Later executions just reuse it with different values (fast for repeated queries).
        
c. **Cleaner Code**
    - Easier to manage queries with multiple inputs.   

> [!Note] Each action that is to be performed to the database should be on its own script. ie userdelete.php, userupdate.php , formhandler.php etc
