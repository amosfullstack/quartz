Referrence types and Objects are used to Store Complex Values.

There is a Class in java called <span style="color:rgb(255, 255, 0)">Point</span>  it is found in <span style="color:rgb(255, 255, 0)">java.awt.*</span> 
- This point can be used to store the x and y coordinates
ie:
```java
Point pointA = new Point(10,10);
Point pointB = new Point(3,6);
```
- A new point object called pointA of the Point class has been instantiated.
- If you are to write a Game, this class is very useful.
- pointA has stored complex data, that is x and y coordinates.
- There are many methods in the point class such as
    .move()
    .getLocation()
    .distance()
  ie:
```java
System.out.println(pointA.distance(pointB));
//Finds the distance between two points

pointA.move(10,11); //This will movw the point Up, if it is a snake game for example.
```



QN:
1. Is Java Passed by value, or passed By Reference?
- Java is passed by Value.
ie:
```java
int age = 21;
int newAge = increment(age);

//This two new ages do not affect each other when the method increment is called.
//the age is just stored in newAge
```


2. When working with Refference Data Types, the change in one value, will lead to change the other.
ie:
```java
Point pointA = new Point(10,10);
Point pointB = pointA;

pointA.x = 100; //This will change poin A and B
pointB.y = 90;  //This will also change point A and B
```
Why?
- This is because both pointA and B references are both pointing to the Exact same object.
- If you change point B, it will update the referenced object.

SO DONT CONFUSE:
i. The Concept of <span style="color:rgb(255, 255, 0)">Passing by Value</span> .
ii. The Concept of <span style="color:rgb(255, 255, 0)">Passing by Referrence</span> .


Generally : Java is Pass by Value for both Refference types and Primitive types.
