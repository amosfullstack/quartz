## Overview
- Interfaces allows us to archive Polymorphism

What is Polymorphism?
- Polymorphism mean many forms.


- The Classes below will need the same methods.
ie:
```java
  
    //Polymorphism = many forms  
    //Car - move,break,getCurrentSpeed    
    //Bicycle - move,break,getCurrentSpeed 
     //Electric Scooter - move,break,getCurrentSpeed
     //All are Vehicles
```

<span style="color:rgb(255, 148, 148)">Warning</span> : We Could create a vehicle class with all the methods, but the way that Car works is different from how the Bicycle Works.
- Even thought the Methods are the same, but the internals are different.
So, it is best to use <span style="color:rgb(0, 176, 80)">interface</span> . 


###### Interface
An Interface can contain three things.
i. Static immutable fields / also constants 
ii. Abstract Methods 
iii. Default Methods

ie: 
```java
public interface Vehicle{

 void foo(); //The methods are always public and abstract
 void bar(); // Do not start with public, may result to duplication

}
```

- We use <span style="color:rgb(255, 255, 0)">implements</span> keyword to include the interface in the class
ie:
- It is compulsory to override them.
```java
public class Car implements Vehicle{
  
  @Override
  public void foo(){
    //Our Own Logic
  }
  public void bar(){
    //Our Own Logic
  }
}
```


i. We can use the interface to replace multiple Classes declaration.
- Supose we want the Car,Bicycle and ElectricScooter classes to used in the Person Class as datatypes.
- But,  he Car,Bicycle and ElectricScooter classes extends the Vehicle Interface.

Instead of: 
```java
public class Person{
  private Car[] cars;
  private Bicycle[] bicycles;
  private ElectricScooter[] scooters;
}
```
- This makes code too duplicate.

Just use:
```java
public class Person{
  private Vehicle[] vehicles;
}
```
- The interface instantiates all classes at once.
- No need to declare each class separately

Now inside the Main function
```java
Car car1 = new Car(); //Instantiate car object
Bicycle bicycle1 = new Bicycle(); //Instantite the bicycle object
Vehicle[] vehicles = {car1,bicycle1} //Instantiate the Vehicles objects of interface

Person person1 = new Person("John",vehicles); //John has vehicles 

```


ii. We can have <span style="color:rgb(255, 255, 0)">constants</span> in interfaces
ie:
```java
public interface Vehicle{
  //constants
  double PURCHASE_RATE;
}
```
- by default it is public and not otherwise


iii. Default methods in interfaces
- The default methods overwrites the abstract property.
- The methods are not compulsory to implement.
ie:
```java
public interface Vehicle{
 //default method
 default double milestoKm(int speed){
   return speed*1.609;
 }
}
```
