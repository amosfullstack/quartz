What are Exceptions?
- An <span style="color:rgb(255, 255, 0)">exception</span> is an event which occurs during the execution of the programme that distrupts the normal flow execution of programme instructions.
- Exceptions are errors that occur during the runnig of the programme.
- Exceptions should be correctly handled to prevent the crashing of the application.
- We use <span style="color:rgb(255, 255, 0)">try</span> and <span style="color:rgb(255, 255, 0)">catch</span> block to handle the exception.


Practicals :
i. How to Handle <span style="color:rgb(0, 176, 80)">Specific</span> Exception.
Example 1 : 
```java
for (int i=10; i>=0; i--){  
    System.out.println(10/i);  
}
```
The code above:
> Throws an exception ie:
> <span style="color:rgb(255, 148, 148)">java.lang.ArithmeticException</span> division by zero is not possible (/ by zero) 



Example 2 :
```java
int number = Integer.parseInt("1x");  
System.out.println(number);
```
Also, the code above:
> Throws an exception ie:
> <span style="color:rgb(255, 148, 148)">java.lang.NumberFormatException</span> the input can not be parsed because its a String

<span style="color:rgb(0, 176, 80)">Solution</span> : Try and Catch the Exception, we do not want our programe to be distrupted duting execution.
ie :
```java
try{  
    int number = Integer.parseInt("1x");  
    System.out.println(number);  
}  
catch (NumberFormatException e){  //Get the name of the exception from the error message 
    System.out.println("Failed to parse 1x");  
}
```


Example 3:
- Catching more than one Exception
- You can have as many catchblocks as you want, there is no limitation.
ie:
```java 
try{  
    int number = Integer.parseInt("1x");  
    System.out.println(number);  
  
  
    for (int i=10; i>=0; i--){  
        System.out.println(10/i);  
    }  
}  
catch (NumberFormatException e){  //Exception one
    System.out.println("Failed to parse 1x");  
}  
catch (ArithmeticException e){   //Exception two
    System.out.println("Can not Divide by Zero");  
}
```


ii. How to Handle Exceptions that are <span style="color:rgb(0, 176, 80)">not Specific</span> .
- There are scenarios where you dont know the exact exception that will be thrown.
- Use <span style="color:rgb(255, 255, 0)">e.getMessage()</span> to get the message of the exception that was thrown .
```java
catch (ArithmeticException e){   
    System.out.println(e.getMessage());  
    
    e.printStackTrace(); //Prints all that gibrish about the errors on the console.
}

```

NB : Appart from e.getMessage() there other methods you can use. Read the Documentation to Explore them.


QN : What to do if you do not know ahead of time, the exception that will be thrown?
<span style="color:rgb(0, 176, 80)">ANSWER</span> : In that case , we are going to use just <span style="color:rgb(255, 255, 0)">Exception</span> without using its exact name.
ie:
```java
catch (Exception e){   
  //Catch All Exception
    System.out.println(e.getMessage());  
}
```


iii. <span style="color:rgb(255, 255, 0)">finally</span> block
- The finally block will always excecute regardless.
- This is maily used for cleanup purpurses.
- Where the business logic throws an exception or not., the finally code block will always run.
- Example : The Database connection.
ie:
```java
catch (Exception e){   
  //Catch All Exception
    System.out.println(e.getMessage());  
}
finally{
  sout("Finally always Runs");
}

```
