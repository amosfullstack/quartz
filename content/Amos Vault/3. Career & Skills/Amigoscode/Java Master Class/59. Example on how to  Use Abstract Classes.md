## **Scenario**
- Consider we have a super class with common functionality.
- We should have an abstract method so that the subclass should just change a slight implementation.

![[abstract classes.png | 400]]


###### Key Points to Remember

Abstract Classes
- Can not be instantiated.

Abstract Methods
- have no body
- need to be inside the abstract classes or interfaces

Avoids code duplication and improves readablily.

The purpose of the abstract class is to function as the base for subclasses

Encapsulate some common functionality in one place and let sub classes  implement differences.
