#### Checked Vs  Unchecked Exceptions
 
i. Unchecked Exception (Runtime Exception)
- These are exceptions that can be thrown during the normal operation of the Java Virtual Machine.
- Runtime exception is the superclass of the exceptions that can be thrown during the noraml operation of the Java Virtual Machine.
 ie : The application is running just fine , then at some point it throws an exception.

ii. Checked Exceptions (Compile Time Exception)
- The Class Exception and and its classes that are not also subclasses of RuntimeException  are checked exceptions.
- The exception should be dealt with Even before compiling of the code.
Example : In dealing with files.
```java
File file = new File("src/file1.txt");  
if(!file.exists()){  
    file.createNewFile();  
}

//Error
java: unreported exception java.io.IOException; must be caught or declared to be thrown
```
> The above code will not run. And it will not Compile.
> - The Error means that the IOException should be handled first.

<span style="color:rgb(0, 176, 80)">Solution</span> : The try catch block is compulsory for checked exceptions.
ie :
```java
try{  
    File file = new File("src/file1.txt");  
    if(!file.exists()){  
        file.createNewFile();  
    }  
}catch (IOException e){  
    System.out.println(e.getMessage());  
}
```



#### How to throw the Exception.
- Throwing an exception just mean splacing the exception anywhere yuou want in the code.
 - Syntax to throw an exception
 ```java
 throw new <exception type> ("Message");
 
 //ie
 
 throw new ArithmeticException("Can not Divide by 0");
 ```

Example : Throwing an exception, when the wrong argument is passed.
```java
public static double divide(int a,int b){  
    if(b==0){  
        throw new IllegalArgumentException("Cannot Divide by 0");  
    }  
    return a / b;  
}
```
