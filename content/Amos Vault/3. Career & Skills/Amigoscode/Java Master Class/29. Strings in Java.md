#### How Strings are Stored in memory.
- This is very important if you want to peform comparison on strings.
- Strings are refrerence types.
- ie:
```java
String name1 = "Jamila";
String name2 = "Jamila"

//These two varables point on the same location in memory
```
> That means both name1 and name2 , use one jamila that is in memory.
> name2 does not create another jamila.
> String Pool is used : if the varible is declared it looks in the spring pool in the heap of                                               memory to see   if the value exists. If it exists it just reuses it.

With String Pool, the memory is Optimised. No random objects are created on the garbage collector.

#### Strings are Immutable Objects.
- This means that the value of the String never changes.
- Once  you define a string, the value never changes, What changes realy is the refference.

ie: 
```java
String name01 = "Jamila";
String name02 = "Jamila";

//Trying to change.
```
