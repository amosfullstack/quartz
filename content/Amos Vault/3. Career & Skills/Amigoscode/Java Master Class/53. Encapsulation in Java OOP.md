What is Encapsulation?
- The process of binding an object state and behaviour together in one unit.
- Wrapping data and code acting on that data together.
- We can make the class attribute hidden from other classes using encapsulation.
- Prevent classes from being tightly coupled.
- ie : We can easily modify inner workings of a class without affecting the rest of program consumers 
- We can make our application Robust to changes.
- We have full control : Prevent the attributes to be modified directly.(direct access is not allowed)
 Eg: Allowing the user to edit the bank balance.

NB : Most people confuse, encapsulation is not the process of hiding Data.

![[Encapsulation.png | 300]]

Therefore : Classes are Capsules that bundle tha attributes and methods together.


For Encapsulation :
- The attributes should use the private access modifier. This prevents direct access to the attributes.
- Use the getters and Setters to access the attributes.


NB : Through encapsulation we can hide data. But Encapsulation is not the process of hiding data.
