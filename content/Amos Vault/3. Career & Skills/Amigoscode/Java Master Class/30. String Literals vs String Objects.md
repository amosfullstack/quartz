- In order to understand String Equality, We have to understand string literal vs String object.

<span style="color:rgb(255, 148, 148)">i. String Literal</span>

```java
String name01 = "Amos";
```
- The above syntax of defining the string  is called the String literal.
- It creates the string within the string pool.

ii. String Object
- This is another way of defining a string.
```java
String name01 = new String("Jamila");
```
- The string object that is created with the above syntax is kept in the HEAP and not in the string POOL.
- The Object in the HEAP and the Object in the POOL are completely diffrent, even though they have the same value.
- <span style="color:rgb(255, 148, 148)">Therefore</span> : When we use the new keywork, we have no control of whether the value can be reused or not.

###### Comparing Strings. 
- After leaning String Literals and String Objects, We can now learn how to compare Strings

```java
String name1 = "Jamila";
String name2 = "Jamila";
String name3 = new String("Jamil");

sout(name1 == name2); //true
sout(name1 == name3); //false
```

- The double equal sign == , compared the references. hence both name1 and name2 point to the exact same reference(Reference to exact same object).


So, THE RIGHT WAY . To perfom equality on two given Strings.
- use <span style="color:rgb(0, 176, 80)">.equals</span> method.
- Using .<span style="color:rgb(255, 255, 0)">equals</span> means we are comparing the content within the variables.
ie :
```java
sout(name1.equals(name2)); //true
sout(name1.equals(name3)); //true
```

NB : If you want to compare strings, never use double equals == . Instead use .<span style="color:rgb(255, 255, 0)">equals</span> .
