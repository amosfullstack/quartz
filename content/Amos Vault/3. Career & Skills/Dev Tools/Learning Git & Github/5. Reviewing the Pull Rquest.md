- The Pull Requests has to be Reviewed before merging to the main brach.

Steps:
i.  Click on Pull Requests Tab:
ii. Click the pull Request It self to see its details

iv. Converstations(tab) : See the converstaions that have been made about the pull requests.

v. Commits(tab): See details of the specific commits made.

vi. Files Changes(tab): See all the files in which the changes have been made.

v. When you are confident about the pull request changes you are ready to merge the pull request.

vi. When you merge the pull request, the changes on your branch are merged with another branch.

v. Click conversations, and then Merge. You will receive the success message of the merged branch.
ie:
```
### Pull request successfully merged and closed

You're all set — the [mybrach](https://github.com/amosfullstack/learningGithub001/tree/mybrach) branch can be safely deleted.
```
