###### Steps 

i. Install git.
```bash
sudo apt install git
```

ii. Check if git is installed
```bash
git -v
```

iii. Configure your username.
```bash
git config --global user.name "amosfullstack"
```

iv. Configure your email
```bash
git config --global user.email "amos@gmail.com"
```

Check your global configuration
```bash
git config --list
```

v. Intergrate your git with github account.
- Clone the repository
```bash
git clone  <repo-link https>
```


<span style="color:rgb(255, 0, 255)">Addition</span> :
vi. Configuring the default editor to vs code(Optional) 
```bash
git config --global core.editor "code --wait"
```

Vii. The global setting are stored in a file, and you can edit them at once by opening the file.
```bash
git config --global -e
```
- This will open the file in the default editor, in our case is vim.

viii. Configure how git should handle end of lines.
- In windows, end of line is marked with two special characters , <span style="color:rgb(255, 255, 0)">carriage return</span> `\r`  and  <span style="color:rgb(255, 255, 0)">Line Feed</span> `\n`
- In mac/Linux Systems end of line are indicated with<span style="color:rgb(255, 255, 0)"> line feed</span> `\n`
- If we dont handle end of lines properly. we are going to run in some issues dwon the road.
- So, We have to configure the property called <span style="color:rgb(255, 148, 148)">core.autocrlf</span> which is short for carriage return line field
Example : Two people working on the repo one using a mac and on windows.

On windows.
```bash
git config --global core.autocrlf true
```

On mac/linux
```bash
git config --global core autocrlf input
```
