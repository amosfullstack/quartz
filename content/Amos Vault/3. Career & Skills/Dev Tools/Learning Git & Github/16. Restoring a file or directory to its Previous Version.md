## Overview
- When git tracks a file, it stores evry version of that file in its database.
- That means, if we screw things. We can easily Restore a file or a directory to a previous version.

#### Practical
i. Delete a file from both a working directory and the staging area.
```bash
git rm file1.js
```

ii. Make a commit for the deleted file
```bash
git commit -m "Deleted file1.js"
```

Solution : To restore the file, we can restore or undo the last commit. (We shall see this later)

iii. We want to restore the file to its previous version.
- Find the file in the logs
```bash
git log --oneline
```

iv. Restore the file
```bash
git restore <source fo the file> <specify full path to the file>

ie: 
git restore --source=HEAD~1 file1.js
```
- So the file will be restored as untracked.

So, this is how we can <span style="color:rgb(255, 0, 255)">restore</span> a file to a previous version.
