#### Overview
I faced the error when trying to push  local files to the github repository, I also faced the same when trying to pull the remote repos.
ie:  <span style="color:rgb(255, 148, 148)">ERROR BELOW</span>
```bash

git clone git@github.com:amosfullstack/learningGithub001.git
Cloning into 'learningGithub001'...
git@github.com: Permission denied (publickey).
fatal: Could not read from remote repository.

Please make sure you have the correct access rights
and the repository exists.

```

<span style="color:rgb(255, 148, 148)">Cause</span> :
- The error shows that GitHub no longer supports normal password authentication. You need to use either:

###### Option 1: Use SSH (Recommended)

<span style="color:rgb(0, 176, 80)">It is better to use the Recommended method..</span>

i. First Set the remote Origin url to the SSH link. (Copy it from github)
ie:
```bash
git remote set-url origin git@github.com:amosfullstack/learningGithub001.git
```

ii. Generate SSH Key
```bash
ssh-keygen -t ed25519 -C "amosfullstack@github.com"
```
- Press Enter to accept default location
- Optional: Add a passphrase for extra security
Result:
```bash
Your identification has been saved in /home/amos-dev/.ssh/id_ed25519
Your public key has been saved in /home/amos-dev/.ssh/id_ed25519.pub
The key fingerprint is:
SHA256:9J/RmfTRzaBtvj5/kmh1zpQJQFzg1R27+G0O6Xz1A5E amosfullstack@github.com
The key's randomart image is:
+--[ED25519 256]--+
|          ooooo.o|
|          .o.o +=|
|        .  .o =o+|
|       . .   E.+o|
|        S . ..B.+|
|           . +o==|
|            oo=B=|
|            o++=*|
|           .  +==|
+----[SHA256]-----+
```

iii. Start SSH Agent and Add Key
```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

iv. Copy Your Public Key
- Copy the entire output (it should look like: `ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAI...`)
```bash
cat ~/.ssh/id_ed25519.pub
```

RESULT:
```bash
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIJJLTPBZRcz77ZJRxn94A8A9mL1HvP4rYPQKkEKGtGM+ amosfullstack@github.com
```

v. Add SSH Key to GitHub
- Go to [GitHub SSH Keys Settings](https://github.com/settings/keys)
- Click "New SSH key"
- Paste your public key
- Give it a title (e.g., "My Laptop")
- Click "Add SSH key"

vi. Test Connection
```bash
ssh -T git@github.com
```
RESULT:
```bash
Hi amosfullstack! You've successfully authenticated, but GitHub does not provide shell access.
```

Meaning: 
**GitHub does not provide shell access** - This is normal! GitHub doesn't allow you to SSH into their servers like a regular server. Git operations (push, pull, clone) will work, but you can't get a terminal shell on GitHub's servers.

ie: Shell access wild be like this
```bash
# This is what you CANNOT do with GitHub:
ssh git@github.com
# Then run server commands like:
ls -la /home/git/repositories/
cd /var/log/
cat auth.log
top  # See server processes
vim /etc/config  # Edit server files
```

vii. Push your cahnges
```bash
git push -u origin master
```
###### Option 2: Use Personal Access Token (PAT)

<span style="color:rgb(0, 176, 80)">If you prefer using HTTPS...</span>

i. First set the remote origin url to use https link
ie:
```bash
git remote set-url origin https://github.com/amosfullstack/learningGithub001.git
```

Then, push the code:
```bash
git push -u origin master
```

- Go to GitHub.com → Settings → Developer settings → Personal access tokens → Tokens (<span style="color:rgb(0, 176, 80)">classic</span>)
- Generate new token with appropriate permissions (repo scope)
- Copy the token
- Use the Token as your password.
ie:
```bash
username : amosfullstack
password : use PAT classic generated from github
```

NB: Configure Git <span style="color:rgb(255, 0, 255)">credential helper</span> to store token
ie: 
```bash 
# Store your credentials
git config --global credential.helper store

# Then push (it will ask for username and password/token once)
#then push code
```

<span style="color:rgb(255, 148, 148)">BUT REMEMBER</span> : When the url is changed to SSH url, then this http method wont work, so refer to the <span style="color:rgb(255, 0, 255)">Option one procedure</span> .
