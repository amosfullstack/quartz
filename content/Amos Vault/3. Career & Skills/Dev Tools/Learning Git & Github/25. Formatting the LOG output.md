## Overview
- If you dont like the default output of the log command, you can easily customize the output as you wish.

Practical :
i. Customizing the log ouput
```bash
git log --pretty=format:"%Cgreen%an%Creset committed %H on %cd"

//%an -> author name displays
//committed -> the commit id displays
//%H -> Display the hash of the commit
//%h -> Make the hash shorter
//%cd -> Display the commit date
//%Cgreen ->Change the color t green
//%Creset -> Stop displaying the color green
```
- You cant claim all this, Refer to the documentation if you need to.


#### Create Aliases

Practicals:
i. Creating an <span style="color:rgb(255, 255, 0)">Alias for custom log</span> output.
- In git we can create the Aliases for the commands we use frequently, so we do not have to type them in a long form.
ie:
```bash
git config --global alias.lg "log --pretty=format:'%Cgreen%an%Creset committed %H on %cd'"
```

So: Our custom command becomes.
```bash
git lg
```

ii. Creating an Alias for Easily unstaging the changes.
ie:
```bash
git config --global alias.unstage "restore --staged ."
//This will restore all the files in the staging area
```

So : The custom command becomes
```bash
git unstage
```
