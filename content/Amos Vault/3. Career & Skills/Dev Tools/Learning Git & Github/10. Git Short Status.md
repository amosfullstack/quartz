We have already seen how to see the status of the working directory and the staging area using the status command.
ie:
```zsh
git status
```

But, But the output of this command has many words.

Alternative :
```bash
git status -s 
// Use the short statys flag
```


#### Practical
i. Modify one of the existing files
```bash
echo sky >> file1.js
```

ii. Add a new file 
```bash
echo sky > file2.js
```

iii. Look at the status.
```bash
git status
```
- The ouput of this command has many words to read

iv. Lets use a   -s flag
```bash
git status -s

//Output
 M  file1.js //Red M to the right column
?? file2.js

```
- The output of this command has few words and it is much easier to read.
> There are two colums before the file1.js, <span style="color:rgb(255, 148, 148)">M </span>says that there are modifications in the working directory.
  <span style="color:rgb(255, 148, 148)">??</span> For file two, signifies that its a new file to both working directory and staging area.

v. Now let add file1.js
```bash
git add file1.js

//Output on status
M  file1.js  //Green M to the left column
```
> The modifications are now added to the staging area


Generally for short status.
M : Modified
? : New file not added
A : Added
