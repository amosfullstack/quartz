QN : What if we have changes in the working directory that we just want to throw away.?
Answer : We can discard the local changes using the <span style="color:rgb(255, 255, 0)">restore</span> command.

ie: 
i. Restore <span style="color:rgb(255, 0, 255)">new</span> the file from the staging area to our working directory
```bash
git restore file1.txt

git restore . // This restores all

NB : This command does not affect untracked files. ie : Files with ??
```

ii. To remove <span style="color:rgb(255, 0, 255)">new</span> untracked files from or working area
- We use the <span style="color:rgb(255, 255, 0)">git clean</span> command
ie:
```bash
git clean
//use flags
-f //Delete forcefully
-d //Remove whole directories

git clean -fd
```



<span style="color:rgb(0, 176, 80)">Addition : 3/09</span>

iv .  Removing add files from the staging area.
- What if we want to <span style="color:rgb(255, 255, 0)">undo the git add . </span>command.
- We want to remove all the files recursively that have been added to the Staging area .
Soln :
```bash
git rm --cached . -r

//This undoes everything recursively that has been done by git add .
```
