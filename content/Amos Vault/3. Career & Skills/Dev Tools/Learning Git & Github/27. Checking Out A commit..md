## Overview.

What is Checking Out a commit?
- Checking Out a commit involves <span style="color:rgb(255, 255, 0)">viewing a complete snapshot </span>of our project at a particular point of time.
- This is like <span style="color:rgb(255, 255, 0)">loging in</span> in the previous version of our project.
- So, Our woking directory will move back in time.
- This is like time Travelling to the Past.

Practical :
i. Checking out a particluar commit
```bash
git checkout <commit id>

//ie
git checkout dad47ed
```

Waring says :
You are in 'detached HEAD' state.
- This means the head pointer has moved.
- Normal, the HEAD pointer points to our currently working directory

<span style="color:rgb(0, 176, 80)">Before commit checkout.</span>
ie: Image of HEAD pointer pointing to the master branch.

![[head pointer pointing to master.png]]

<span style="color:rgb(0, 176, 80)">After Commit Checkout :</span> 
- Image of the headpointer in Detached State.
- ie : The head Pointer moves to the commit that we have Checked out
![[head pointer moves to a commit the we have checked out.png]]

- So, now the head is pointing to a specific comit.
- In this situation, we should <span style="color:rgb(255, 255, 0)">not</span> create new commits. 
- <span style="color:rgb(255, 148, 148)">Because</span> : The commit will not be traceable, it will be a <span style="color:rgb(255, 255, 0)">dead commit</span> . Git wil eventually delete it to save Space.
- <span style="color:rgb(255, 148, 148)">What to do</span> : We should only look arround and only experimental Changes.


ii. To see all the Commits while in a git chekout,all the extra
```bash
git log --oneline --all
```
- You can observe where the head pointer is pointing now.

iii. To attach the head Pointer to the master Branch
```bash
git checkout master
```
- Not we are on the master branch.
- We can now make new commits.
