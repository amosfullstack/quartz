What is <span style="color:rgb(255, 0, 255)">Git</span>?
- Git is a version control(Source Control) system, which helps to manage tha changes that you or your collegues have made to the files over time.
- With git you are able to see the history of your files and recover the old version if needed.
- If we mess up ou code, we can easily revert the changes to an ealier state.
- With version control system, we can track our history and work together.

<span style="color:rgb(255, 148, 148)">A Branch</span> : A branch is a copy of your source code where you can work without changing the original branch.
- This is useful when you want to work on the new feature of your project.
- It also useful when you want to solve it back.
- After working on the branch you can synchronorize it with the original one or other branches. (This operation is called <span style="color:rgb(255, 255, 0)">Merge</span>)
- There is no limit on the number of branches you can create during the development process.
- You can move your changes beackwards and foward, if you need.

What is Github?
- Github is an online service where developers can work together.
- It is a cloud-based storage service to store code and share it with others.

#### A Tour on Github

i. <span style="color:rgb(255, 148, 148)">Readme File</span> : 
- This file Explains the purpose of the project.
- The quick starter, the basic documentation for contributors.
- And anything that is helpful for the contributors to know.
- If you make a good read me file you have a better chance of your project getting contributtors. Because it is like a business card.

ii. <span style="color:rgb(255, 148, 148)">Issues Tab</span> :
- Issues tab are used to track everything related to the project, such as bugs and feature requests.
- Issues can be assigned to a specific user.
- You can also start a disscusion on the issue to explain the details of the new feature request or describe a bug.
- Issues is like a chatting place.

iii. <span style="color:rgb(255, 148, 148)">A Pull request</span> :(tab)
- A pull request is a change, such adding,changing or deleting files.
- With pull requests, you can f<span style="color:rgb(255, 255, 0)">acilitate code reviews</span>. Modified code is shown in the project  files.

iv. <span style="color:rgb(255, 148, 148)">Github Action</span> :(tab)
- An action is something tha can be <span style="color:rgb(255, 255, 0)">triggered automatically </span>or not.
- Yo can use an action to <span style="color:rgb(255, 255, 0)">automate deployment</span> for instance.

v. <span style="color:rgb(255, 148, 148)">Projects</span> : (tab)
- The projects tab allows you to view and see your project organisation in a kanban card-style view.
- You can not change the code from the projects tab. But you can manage the projects as the project manager.
- The status of the issues  and the backlog are displayed.

vi. <span style="color:rgb(255, 148, 148)">Wiki</span> : (tab)
- The wiki tab is for documentation.
- In many projects, the users document the project roadmap, the next feature release, or other information.
- You can use markdown to format this pages.

vii. <span style="color:rgb(255, 148, 148)">Security</span> : (tab)
- A security tab is used to manage the security policies of the project eg(access, public view or private etc..)

viii. <span style="color:rgb(255, 148, 148)">Insights</span> : (tab)
- Insights tab, allow you to see the insights of all activities of the project.
- This may include authors, total commits, total pull requests.
