- Wht if  we Want to view what Exactly we have changed in a given commit.
<span style="color:rgb(0, 176, 80)">Answer</span> : Use the <span style="color:rgb(255, 255, 0)">git show</span> command.

ie :
- Reference a commit by its unique identifier.
```bash
git show <commit identifier>
ie:
git show d9dopj90
```

ie :
- Reference a commit by using the number of steps from the head
```bash
	git show HEAD~1 (First commit from the head)
```

ii. To see the <span style="color:rgb(255, 255, 0)">exact final version</span> of a file stored in a certain commit
- Specify the file path
ie: 
```bash
git show HEAD~1:<file path>

Example
git show HEAD~1:.gitignore

or
git show HEAD:./filex.txt
or
git show HEAD:filex.txt

or
git show HEAD~2:section/snapshots/filey.txt
```

iii. If we want to see <span style="color:rgb(255, 255, 0)"> only the files </span>that have been modified in a certain commit. We do not want to see the contents of the files.
```bash
git show HEAD~2 --name-only

//HEAD~2 : Means move back two steps
```

iv . To see what has happened to the file. Either(deleted or modified).
```bash
git show HEAD~2 --name-status
```

v. To see all the files and directories in a commit
```bash
git ls-tree
```

- Using the git show command we are able to see the objects that are in the git database
- Example:
- Commits
- Blobs(files)
- Trees(directories)
