## Overview
- In Actual Repositories we have thousands of Commits, depending on the age of the Repository.

- We can filter the history by : (author, by date, by message, by content etc)

###### Practicals
i. Viewing the <span style="color:rgb(255, 255, 0)">last three</span> commits.
```bash
git log --oneline -3
```

ii. Fitering the commit by Certain <span style="color:rgb(255, 255, 0)">Author</span>
```bash
git log --oneline --author="Mosh"
```
- All the commits that mosh submitted will be seen

iii. Filtering the Commits by <span style="color:rgb(255, 255, 0)">date</span> .
```bash
//See all the commits created on this date and before
git log --online --before=""

//See all the commits created on this date and after
git log --oneline --after="2020-08-17"

git log --oneline --after="yesterday"

git log --oneline --after="six years ago"
```

iv. Fiter then commit with a certain <span style="color:rgb(255, 255, 0)">word</span>  in the message.
Eg : Filter the commits that have the word <span style="color:rgb(255, 255, 0)">gui</span> in the message.
     : The searched word is case sensitive.
```bash
git log --oneline --grep="gui"
```

v. Filter the commits by <span style="color:rgb(255, 255, 0)">content</span> .
Eg : Filter all the commits that have added function declaration.
```bash
git log --oneline -S"hello()"
```

vi. To see the actual changes, you can add the patch option.
```bash
git log --online -S"hello()" --patch
```

vii. Filtering the commits by a <span style="color:rgb(255, 255, 0)">range of commits</span> .
Eg : We want to view just the commits in a cretain range.
```bash
//Use the staring and the ending commit ids
//Start from down to top
//ie: From earliest to latest
git log --oneline fb0d184..edb3594
```

viii. Findind all the commits that have <span style="color:rgb(255, 255, 0)">touched a particular file</span> .
Eg : Finding all the commits that have modified the toc.txt file(table of contents)
```bash
git log --oneline toc.txt


//If git complains about the file, include the -- before the filename
git log --oneline -- toc.txt
```

ix. To the the actual changes in the commits, use the patch option again
```bash
git log --oneline --patch --toc.txt 
//Use the patch option before the file name.
```
