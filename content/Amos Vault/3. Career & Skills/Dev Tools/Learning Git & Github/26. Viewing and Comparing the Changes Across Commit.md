## Overview
- We have already seen how to compare staged and unstages files using the diff command.
- We shall use the same command to compare the changes between to commits.

Practical :
i. Comparing changes between two commits
This command shows you the **difference between the commit from two steps ago and the very latest commit**.

Breakdown:
- **`HEAD~2`**: The commit that is two commits before the current one
- **`HEAD`**: The very latest commit (most recent)
- So this shows you **all changes made in the last two commits**
ie:
```bash
git diff HEAD~2 HEAD
```

ii. f you want to see the changes in a <span style="color:rgb(255, 255, 0)">particular file </span>between two commit.
```bash
git diff HEAD~2 HEAD file2.txt
```

iii. If you want to see the<span style="color:rgb(255, 255, 0)"> list of files</span> that have been changed in the commits
```bash
git diff HEAD~2 HEAD --name-only
```

iv. If you want to see the <span style="color:rgb(255, 255, 0)">list of the files </span>and the <span style="color:rgb(255, 255, 0)">type of change</span> that has been done.
```bash
git diff HEAD~2 HEAD --name-status
```
- If you have modified, new, or deleted files, they will be listed here.
