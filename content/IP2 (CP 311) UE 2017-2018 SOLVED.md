**a) What is a Document Type Definition (DTD)? What DTD do you generally use? Why? What are its Pros and cons.**

> [!Tip]- Answer (Click to show)
> **Document Type Definition (DTD):**
>
> A Document Type Definition (DTD) is a set of markup declarations that define the structure, elements, and attributes of an XML document. It specifies the rules for the document structure, including:
> - Which elements can appear in the document
> - The order and nesting of elements
> - Which attributes are allowed for each element
>
> **Commonly Used DTD:**
> Generally, **XHTML 1.0 Strict DTD** is commonly used because:
> ```dtd
> <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN"
> "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
> ```
>
> **Why:**
> - Enforces clean, well-structured code
> - Separates content from presentation
> - Ensures cross-browser compatibility
> - Promotes accessibility and best practices
>
> **Pros of DTD:**
> - **Validation**: Ensures documents follow defined rules
> - **Standardization**: Creates consistent document structure
> - **Sharing**: Allows sharing of document structures across organizations
> - **Legacy Support**: Supported by many older systems
> - **Namespace Control**: Defines which elements and attributes are allowed
>
> **Cons of DTD:**
> - **Limited Data Types**: Only basic string data types
> - **Non-XML Syntax**: Uses its own syntax, not XML
> - **No Namespace Support**: Limited support for XML namespaces
> - **Less Precise**: Cannot define specific data constraints like numbers, dates
> - **No Inheritance**: No support for element inheritance
>
> **Alternative:** XML Schema (XSD) is now preferred as it addresses many DTD limitations, using XML syntax and supporting data types, namespaces, and inheritance.

**d) Explain two advantages of using XML.**

> [!Tip]- Answer (Click to show)
> **Advantages of Using XML (eXtensible Markup Language):**
>
> **1. Platform Independence and Data Portability**
> - XML is text-based and platform-independent, making it easy to exchange data between different systems
> - Can be read and processed by any programming language (PHP, Java, Python, C#, etc.)
> - Ideal for data interchange between heterogeneous systems
> - Example: Web services using SOAP/XML for communication between different platforms
>
> **2. Self-Descriptive and Human-Readable Format**
> - XML uses meaningful tags that describe the data content
> ```xml
> <student>
>     <firstName>John</firstName>
>     <lastName>Doe</lastName>
>     <regNo>CS/001/2018</regNo>
> </student>
> ```
> - Easy to understand without special tools
> - Can be validated against DTD or Schema to ensure data integrity
>
> **3. Separation of Content from Presentation (Additional)**
> - XML stores only data, not how it should be displayed
> - Can be transformed into different formats using XSLT (HTML, PDF, plain text)
> - Same XML data can be presented differently for different devices (mobile, desktop, print)
>
> **4. Extensibility (Additional)**
> - No predefined tags - you create your own
> - Can easily add new elements without breaking existing applications
> - Adapts to changing business requirements
>
> **5. Supports Unicode (Additional)**
> - Can handle multiple languages and character sets
> - Internationalization support
> - UTF-8 and UTF-16 encoding

>

> function isNumber(value) {
>     return !isNaN(parseFloat(value)) && isFinite(value);
> }
>
> console.log(isNumber(42));        // true
> console.log(isNumber("42"));      // true
> console.log(isNumber("42.5"));    // true
> console.log(isNumber("hello"));   // false
>
> // Method 5: Comprehensive function
> function isValidNumber(value) {
>     // Check if it's actually a number and not NaN/Infinity
>     return typeof value === 'number' && !isNaN(value) && isFinite(value);
> }
>
> console.log(isValidNumber(42));           // true
> console.log(isValidNumber(NaN));          // false
> console.log(isValidNumber(Infinity));     // false
> console.log(isValidNumber("42"));         // false
>
> // Method 6: Regular expression for numeric strings
> function isNumericString(value) {
>     return /^-?\d+\.?\d*$/.test(String(value));
> }
>
> console.log(isNumericString("42"));       // true
> console.log(isNumericString("42.5"));     // true
> console.log(isNumericString("-42"));      // true
> console.log(isNumericString("hello"));    // false
> ```

**(e) How can we send mail using JavaScript?**

>
> ```javascript
> // Method 1: Using mailto: protocol (opens default email client)

> function sendMailViaAjax() {
>     // Create XMLHttpRequest
>     let xhr = new XMLHttpRequest();
>     xhr.open("POST", "sendmail.php", true);
>     xhr.setRequestHeader("Content-Type", "application/x-www-form-urlencoded");
>     
>     xhr.onreadystatechange = function() {
>         if (xhr.readyState === 4 && xhr.status === 200) {
>             console.log("Response: " + xhr.responseText);
>             alert("Email sent successfully!");
>         }
>     };
>     
>     let data = "to=recipient@example.com&subject=Test&message=Hello";
>     xhr.send(data);
> }
>

>     let formData = new FormData();
>     formData.append('to', 'recipient@example.com');
>     formData.append('subject', 'Test Email');
>     formData.append('message', 'Hello from JavaScript!');
>     
>     fetch('sendmail.php', {
>         method: 'POST',
>         body: formData
>     })
>     .then(response => response.text())
>     .then(data => {
>         console.log('Success:', data);
>         alert('Email sent successfully!');
>     })
>     .catch(error => {
>         console.error('Error:', error);
>         alert('Failed to send email.');
>     });
> }

>         success: function(response) {
>             alert('Email sent!');
>         },
>         error: function(xhr, status, error) {
>             alert('Error: ' + error);
>         }
>     });
> }
> ```
>
> **Server-side PHP Script (sendmail.php):**

>     $headers .= "X-Mailer: PHP/" . phpversion();
